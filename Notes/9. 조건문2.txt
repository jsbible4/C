1. switch 분기문 
    1) if, else는 조건식이 참, 거짓일 때 두 가지만 처리할 수 있음
    2) else if는 여러 조건을 처리 가능하지만 매 조건식을 지정해야 함
    3) switch 분기문은 조건이 많을 때 유용함 
    4) 예
        switch (값을 판단할 변수)
        {
            case 1:     // 콜라 버튼
                콜라를 내보냄
                break;
            case 2:     // 칫솔 버튼
                칫솔을 내보냄
                break;
            case 3:     // 컵라면 버튼
                컵라면을 내보냄
                break;
            default: 
                제공하지 않는 메뉴
                break

        }
5) switch 분기문은 항상 case 와 함께 사용됨
    - (값을 판단할)변수에 지정한 값이 case에 지정한 값과 일치하면 코드가 실행됨
    - 아무 case에 해당하지 않으면 default의 코드 실행 (default는 생략 가능)
    - case에  break 를 사용하지 않는 경우
        - case를 break로 중단하지 않으면 그 다음에 있는 case, default가 계속 실행됨
    - case 안에서 변수 선언하기 {중괄호 안에서 선언해야 visual studio에서는 에러가 안남.}
        - 중괄호({})로 묶어주면 됨. 

6) switch에서 판별할 수 있는 자료형 
    - 정수 자료형,  문자 자료형 (char) 사용 가능
    - 실수 자료형 ( float, double)은 사용 불가 

2. 조건문 비교

                                       if 문                       switch문
                         =============================================================
    1) 처리가능한 데이터형          char, int, float, double          char, int 
    2) 범위 조건                        처리 가능                     처리 어려움
    3) 두 변수의 비교                    처리 가능                     처리 어려움
    4) 명령코드크기와 실행속도              비효율적                        효율적
